Power & watts diagnostics

Fleet shows CPU package draw (Linux RAPL) plus NVIDIA power.draw when nvidia-smi works. This dialog stays open across page refreshes.

  • Processor energy (RAPL) is exposed under /sys/class/powercap on many bare-metal hosts — not a separate apt package.
  • NVIDIA GPU watts need a working driver and nvidia-smi on PATH for the Fleet process.
  • VMs often lack RAPL; some kernels omit instant power_uw — watts may appear only after the next poll.

Run on the Fleet host (copy as needed):

ls /sys/class/powercap 2>/dev/null | head
for z in /sys/class/powercap/*; do [ -f "$z/name" ] && echo "---" && cat "$z/name" "$z/energy_uj" 2>/dev/null; done | head -n 40
command -v nvidia-smi >/dev/null && nvidia-smi -q -d POWER 2>/dev/null | head -n 40
